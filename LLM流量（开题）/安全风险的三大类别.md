# 一、敏感数据泄露（Sensitive Data Leakage）

### 含义

指在调用 LLM API 的过程中，**用户或系统无意中向模型泄露敏感信息**，而这些信息可能被第三方截获、重放、或误用。

### ⚙️ 发生路径

1. **用户输入层泄露**
    
    - 用户将含有隐私信息的文档、日志、配置等直接发给模型（例如：“帮我总结这份客户名单.xlsx”）。
        
    - 如果调用的API在云端，数据在传输中被加密，但仍可能**被误上传到外部服务器**。
        
2. **日志或缓存层泄露**
    
    - 部分调用框架或SDK会默认记录API请求日志。若日志未妥善保护，敏感信息可能泄露。
        
3. **响应层泄露**
    
    - 模型输出中可能意外“回忆”训练数据中的隐私内容，或输出内部指令、API密钥等。
        
4. **跨系统传播**
    
    - LLM API的响应结果常被其他系统继续处理（如RAG系统、CRM系统），形成“隐性扩散链”。
        

### ⚠️ 实例

|场景|描述|
|---|---|
|公司员工用 ChatGPT 处理内部报告|敏感数据上传到外部API，被模型记录或用于再训练|
|AI客服调用API记录用户对话|聊天内容中含有身份证号或交易记录，被日志存储泄露|
|代码审查辅助工具|上传含有API Key或配置文件的源码给模型，泄露凭据|

### 💡 与加密流量的关系

即使流量被HTTPS加密，**安全边界仍在API的应用层**。  
从流量侧角度，可以检测：

- 上传数据包异常大（文档上传）
    
- 请求频率异常
    
- 长时间会话伴随大体积上行流量
    

这些都是**潜在泄露风险信号**。

# 二、越权访问（Unauthorized Access）

### 含义

指攻击者或非授权用户，**利用 LLM API 的调用接口或身份漏洞，访问或操作原本受限资源**。

### ⚙️ 常见攻击方式

1. **API Key 泄露**
    
    - 攻击者通过网络监听、日志泄露或Prompt注入获取API Key。
        
    - 随后伪装合法用户调用API。
        
2. **身份混淆攻击**
    
    - 攻击者使用代理/中转服务伪装成合法调用源；
        
    - 或利用流量转发绕过访问控制。
        
3. **限流绕过**
    
    - 攻击者通过多IP、代理池或分布式节点绕过调用频率限制。
        
4. **多租户隔离不足**
    
    - 在共享环境中，不同租户的流量未隔离，造成会话或上下文泄露。
        

### ⚠️ 实例

|场景|描述|
|---|---|
|攻击者通过泄露的API Key批量生成文本|造成计费滥用与模型资源消耗|
|第三方插件复用用户token|越权访问了原本仅内部使用的模型接口|
|多用户系统未隔离上下文缓存|用户A能通过Prompt读到用户B的历史会话|

### 💡 从流量侧检测

- 检测API调用来源与凭证模式的异常；
    
- 同一账号在多个地理位置同时调用；
    
- TLS指纹与已知客户端不匹配；
    
- 调用速率或包间隔异常。

# 三、模型滥用（Model Abuse / Malicious Use）

### 含义

指合法API被用于**生成恶意内容**或**触发攻击性行为**，包括违法、攻击性或有害输出。

### ⚙️ 典型滥用类型

1. **恶意内容生成**
    
    - 生成虚假新闻、诈骗邮件、仇恨言论、色情或政治敏感内容。
        
    - 攻击者通过prompt绕过内容过滤（prompt injection / jailbreak）。
        
2. **网络攻击辅助**
    
    - 利用LLM生成钓鱼邮件、恶意脚本、社会工程攻击话术。
        
3. **自动化滥用**
    
    - 攻击者通过批量API请求自动生成大量垃圾内容（spam）。
        
4. **绕过使用限制**
    
    - 使用代理或自建封装API反向调用官方模型，从而逃避监管或计费。
        

### ⚠️ 实例

|场景|描述|
|---|---|
|用户提示模型“帮我生成诈骗短信模板”|模型绕过过滤器生成攻击内容|
|爬虫批量调用 LLM 生成 SEO 垃圾网页|滥用API资源，造成模型过载|
|第三方平台调用模型生成色情图片或对话|法规风险、品牌损害|

### 💡 从流量角度识别

虽然无法看到内容，但滥用流量往往有统计规律：

- 请求间隔短、速率极高（批量生成）；
    
- 请求包体积稳定但响应较大（固定模板prompt）；
    
- 并发连接数异常；
    
- 调用时间段分布异常（例如夜间持续调用）。
    

这些特征可以被加密流量检测模型捕捉。

# 四、总结对比表

|风险类别|本质问题|常见场景|流量侧可检测信号|
|---|---|---|---|
|敏感数据泄露|内部信息误上传到外部模型|文件/日志上传|上行包体大、时延长|
|越权访问|身份凭证滥用、控制缺失|API Key 泄露|异常来源IP、TLS指纹异常|
|模型滥用|模型被用于恶意生成|prompt攻击、批量调用|高频请求、包序列模式异常|
